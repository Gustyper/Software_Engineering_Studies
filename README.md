# Engenharia de Software – 6º Semestre (FGV)

Este repositório reúne meus estudos de **Engenharia de Software** ao longo do 6º semestre do curso de **Ciência de Dados pela FGV**.  
O foco foi entender como escrever código **mais organizado, reutilizável e fácil de manter**, indo além de apenas “fazer funcionar”.

Ao longo do semestre, foram trabalhados principalmente:

- Boas práticas de organização de código  
- Separação de responsabilidades  
- Reutilização e extensibilidade  
- Leitura e entendimento de código legado  
- Aplicação prática de **padrões de projeto**
- Pensamento de software como sistema, não como script isolado

Os arquivos do repositório refletem exercícios feitos em aula e pequenos testes para fixar os conceitos.

## Padrões de Projeto 

Padrões de projeto são **soluções reutilizáveis para problemas recorrentes de design de software**.  
Eles não são código pronto, mas **modelos de estrutura**.

Neste repositório aparecem exemplos como:
- **Factory** – cria objetos sem expor a lógica de criação  
- **Singleton** – garante uma única instância de uma classe  
- **Iterator** – percorre estruturas sem expor sua implementação  
- **Decorator** – adiciona comportamento sem alterar a classe original  
- **Chain of Responsibility / Command** – desacoplam quem solicita de quem executa

## Engenharia de Software

Engenharia de Software é sobre:
- Escrever código que outras pessoas consigam entender  
- Facilitar manutenção e evolução do sistema  
- Reduzir acoplamento e aumentar coesão  
- Pensar em longo prazo, não só na entrega imediata

Este não é um projeto final ou produto.  
É um **repositório de estudo**.
